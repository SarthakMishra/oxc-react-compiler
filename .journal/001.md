## Phase 6: BuildHIR and Remaining Validation Passes

**Date:** 2026-03-11
**Task:** `.todo/build-hir.md -- Full HIR lowering`, `.todo/validation.md -- All remaining validation passes`
**Upstream:** `src/HIR/BuildHIR.ts`, `src/Validation/`

Implemented `build.rs` (~2500 lines) — the OXC AST to HIR lowering pass. The `HIRBuilder` struct manages the current basic block, break/continue target stacks for loop/switch lowering, and a label map for JavaScript labeled statements. Statement lowering covers variable declarations (with destructuring), if/switch/while/for/for-in/for-of, try/catch/finally, throw, return, labeled statements, and expression statements. Expression lowering covers literals, identifiers, binary/unary/logical operators, assignment (including compound assignment), member access (computed and static), call expressions, new expressions, tagged template literals, template literals, JSX (both element and fragment), conditional expressions, optional chaining (converted to explicit null-checks in the CFG), sequence expressions, array/object literals, spread, and function/arrow function expressions. Destructuring of both object and array patterns is fully handled, including defaults, rest elements, and nested patterns. Operator enums are mapped from OXC's AST variants to HIR variants. Nested function expressions are recursively lowered into separate `HIRFunction` values.

Ten additional validation passes were implemented: `ValidateContextVariableLValues` detects mutations of context-injected constants, `ValidateUseMemo` checks argument counts and rejects async callbacks, `ValidateNoCapitalizedCalls` warns on PascalCase function calls outside JSX, `ValidateLocalsNotReassignedAfterRender` detects render-phase variable reassignments, `ValidateNoDerivedComputationsInEffects` warns on derived state calculations inside effects, `ValidateNoSetStateInEffects` detects synchronous setState in effects, `ValidateNoJSXInTry` mirrors the lint rule but operates on the HIR, `ValidateNoFreezingKnownMutableFunctions` detects `Object.freeze(setState)` patterns, `ValidateExhaustiveDependencies` checks for missing useMemo/useEffect dependency array entries, and `ValidatePreservedManualMemoization` checks that the compiler's output is consistent with any manually-written useMemo calls.

## Phase 5: Inference, Reactive Scopes, and Initial Validation

**Date:** 2026-03-11
**Task:** `.todo/inference.md`, `.todo/reactive-scopes.md`, `.todo/validation.md -- Initial passes`
**Upstream:** `src/Inference/`, `src/ReactiveScopes/`, `src/Validation/`

Implemented the full inference pipeline: `InferTypes` performs forward type propagation from instruction kinds (JSX expressions produce `Jsx` type, hook calls produce `Hook` type, etc.); `AliasingEffects` computes per-instruction aliasing effects covering all 40+ `InstructionValue` variants with correct capture/alias/mutate semantics; `InferMutationAliasingEffects` is the orchestration pass that calls `AliasingEffects` and merges effects into the `HIRFunction`; `InferMutationAliasingRanges` converts the per-instruction effects into contiguous mutable ranges per identifier; `InferReactivePlaces` implements a fixpoint propagation starting from known reactive sources (hook calls, props, context) and propagating reactivity through data dependencies until no new places are marked reactive.

The reactive scopes pipeline was implemented across six core passes and a 840-line prune pass: `InferReactiveScopeVariables` uses `DisjointSet` to group co-reactive identifiers into scopes; `AlignScopes` adjusts scope boundaries for method calls, object methods, and block-level alignment; `MergeScopes` collapses overlapping scopes and groups invalidation-coupled scopes; `PropagateDependencies` computes the full dependency set for each scope by walking data flow; `BuildReactiveFunction` converts the HIR CFG into the `ReactiveFunction` tree IR (a nested structure of `ReactiveScope`, `ReactiveBlock`, and `ReactiveInstruction` nodes); `PruneScopes` runs 14 sub-passes including pruning scopes with no memoizable outputs, removing unnecessary scopes wrapping single values, and flattening degenerate scope nesting. The `codegen` pass traverses the `ReactiveFunction` tree and emits JavaScript with `useMemoCache` calls and slot-based cache invalidation.

Three initial validation passes were implemented: `ValidateHooksUsage` walks the CFG detecting hooks called inside conditionals or loops by tracking the depth of conditional/loop terminals; `ValidateNoRefAccessInRender` detects `MemberExpression` accesses on identifiers typed as refs; `ValidateNoSetStateInRender` detects unconditional setState calls in the top-level render path.

## Phase 4: SSA, Optimization Passes, Function Discovery, and Pipeline Skeleton

**Date:** 2026-03-11
**Task:** `.todo/ssa.md`, `.todo/optimization.md`, `.todo/pipeline.md -- Discovery and orchestration`
**Upstream:** `src/SSA/EnterSSA.ts`, `src/Optimization/`, `src/EntryPoint/`

Implemented `enter_ssa.rs` (~1000 lines) using the Cytron et al. algorithm: dominance tree construction via iterative post-order DFS, dominance frontier computation, phi insertion at dominance frontiers for all variables that are live across multiple definition sites, and SSA variable renaming via a recursive dominator-tree traversal with a per-variable definition stack. DIVERGENCE: the dominator computation uses an iterative fixed-point algorithm rather than the Lengauer-Tarjan algorithm used in some upstream tooling, which is simpler to implement correctly and sufficient given the function sizes encountered. `eliminate_redundant_phi.rs` implements iterative redundant phi removal: a phi whose all operands are identical is replaced by that single value, and the replacement is propagated through the entire HIR; the pass repeats until no more phis can be eliminated.

The six optimization passes were implemented: `ConstantPropagation` does forward constant value propagation through the CFG replacing `LoadLocal` instructions with known constant values; `DeadCodeElimination` removes unused instructions (instructions whose result is never referenced) and prunes unreachable basic blocks (blocks not reachable from the entry after a dominator computation); `MergeConsecutiveBlocks` merges pairs of blocks connected by a single unconditional `Goto` edge by appending the successor's instructions and terminal to the predecessor, used to clean up the CFG after earlier transformations; `PruneMaybeThrows` removes `MaybeThrow` terminals that appear outside of try-block scopes where they serve no purpose. `InlineIIFE` and `OptimizePropsMethodCalls` are no-op stubs pending CFG transformation and type inference respectively. The `entrypoint/program.rs` module gained function discovery: an OXC AST walk that identifies React components (PascalCase functions) and hooks (camelCase functions starting with `use`) respecting `CompilationMode` and opt-in/opt-out directives. The pipeline orchestrator in `pipeline.rs` was expanded to call passes in the correct order with config-flag gating.

## Phase 3: HIR Foundation Types and Utilities

**Date:** 2026-03-11
**Task:** `.todo/hir-types.md`, `.todo/environment.md`, `.todo/utils.md`
**Upstream:** `src/HIR/HIR.ts`, `src/HIR/Environment.ts`, `src/HIR/ObjectShape.ts`, `src/HIR/Globals.ts`

Implemented all core data structures for the compiler's internal representation in `hir/types.rs` (~1000 lines). ID newtypes (`BlockId`, `IdentifierId`, `ScopeId`, `InstructionId`) use newtype wrappers over `u32` with `Display`/`From` impls. `Place` pairs an `IdentifierId` with a `ProjectionPath` for member-access lowering. `InstructionValue` is a 40-variant enum covering all lowered expression forms: `LoadLocal`, `StoreLocal`, `LoadGlobal`, `BinaryExpression`, `UnaryExpression`, `CallExpression`, `MethodCall`, `NewExpression`, `PropertyLoad`, `PropertyStore`, `ComputedLoad`, `ComputedStore`, `JsxElement`, `JsxFragment`, `JsxText`, `ArrayExpression`, `ObjectExpression`, `Primitive`, `TemplateLiteral`, `TaggedTemplateExpression`, `LogicalExpression`, `ConditionalExpression`, `Phi`, `StartMemoize`, `FinishMemoize`, and others. `Terminal` is a 20-variant enum encoding all CFG control flow: `Goto`, `Branch`, `Switch`, `Return`, `Throw`, `Try`, `MaybeThrow`, `Sequence`, and labeled variants. `BasicBlock`, `HIR`, `HIRFunction`, and `ReactiveFunction` IR types round out the core structural types. The `ReactiveScope` type contains the scope's dependency set, output set, and mutable range.

`environment.rs` defines `EnvironmentConfig` with ~20 boolean flags governing which passes run and which validations are enabled, plus `CustomHookConfig` for user-defined hook shapes. `object_shape.rs` implements `ShapeRegistry` (a global registry of named shapes), `FunctionSignature` (parameterized call kind and return type), `PropertyShape` (property access shapes), and the `CallKind` enum distinguishing hook calls from regular calls. `globals.rs` implements `register_globals()` which populates the `ShapeRegistry` with shapes for `Array`, `Object`, `Math`, `JSON`, `console`, `String`, `Number`, `Promise`, React hooks (`useState`, `useReducer`, `useEffect`, `useRef`, `useMemo`, `useCallback`, `useContext`, `useId`, `useTransition`, `useDeferredValue`, `useSyncExternalStore`), and DOM globals. `disjoint_set.rs` implements union-find with path compression and union-by-rank. `ordered_map.rs` implements an insertion-order-preserving map using a `Vec` of key-value pairs with `O(n)` lookup — appropriate for the small maps used in scope dependency tracking.
